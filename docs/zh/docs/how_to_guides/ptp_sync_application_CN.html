PTP时间同步使用指导

禾赛激光雷达产品可支持由软件模拟方式获得的PTP主时钟或专用的硬件PTP主时钟设备进行PTP授时。

一、软件模拟方式PTP授时​

通过将使用Linux操作系统的电脑主机配置为PTP主时钟并直连禾赛激光雷达，即可实现软件模拟方式的PTP授时，对应的硬件连接方式可参考下图：

针对不同类型的PTP时间同步协议，在Linux系统中配置PTP主时钟的方法具体如下所述。

1588v2协议​

对于1588v2协议，常用的开源PTP时间同步模拟软件有linuxptp和ptpd（推荐使用linuxptp），对应的操作方法分别如下：

A. linuxptp​

  1. 安装
sudo apt-get install linuxptp ethtool

注：禾赛激光雷达产品支持的linuxptp为v2.0版本，建议在系统中选择相同的linuxptp版本进行安装，以避免ptp4l版本不兼容造成主从时钟的PTP同步失败。

  1. 使用ethtool命令查看当前主机网卡所支持的PTP同步功能
sudo ethtool -T 网卡名

打印信息如下：

注：此处显示的hardware或software表示该网卡具备是否具备硬件时间戳能力，或仅支持软件时间戳功能。其中，硬件时间戳的同步精度能够达到几微秒甚至更低，而软件时间戳的时间同步偏差则一般要于大于硬件时间戳方式。

  1. 运行以下指令，将主机网卡配置为主时钟并启动授时
  • Network Transport -> UDP/IP
sudo ptp4l –m -4 –i 网卡名 –S
  • Network Transport -> L2
sudo ptp4l –m -2 –i 网卡名 –S
  1. 观察命令行打印信息，此时网卡应显示为Grand Master状态

B. ptpd​

  1. 安装
sudo apt-get install ptpd
  1. 运行以下指令，将主机网卡配置为主时钟并启动授时
sudo ptpd -i 网卡名 -C -M
  1. 观察命令行输出信息，此时网卡应显示为PTP_MASTER状态

对应激光雷达端所显示的PTP状态应同步更新为Tracking或Locked（是否为Locked取决于当前主机时间同步精度以及当前激光雷达'Time Offset for Lidar Lock'这一参数设置值）

注：对于多台激光雷达同时进行PTP时间同步的情况，仅需要在网络连接中加入交换机以实现PTP报文的转发，此过程不影响上述PTP主时钟的配置操作。

802.1AS协议​

单台激光雷达PTP授时​

对于单台lidar上使用802.1AS协议进行时间同步，建议使用linuxptp软件工具进行配置，操作方法如下：

  1. 安装（同1588v2协议）；
  2. 使用ethtool命令查看当前电脑主机网卡所支持的PTP同步性能（同1588v2协议）；
  3. 运行以下指令，将主机网卡配置为主时钟并启动授时
sudo ptp4l -S -i 网卡名 -m -f gPTP.cfg

该指令中的gPTP.cfg文件中包含了当前PTP主时钟所需的基本参数配置，具体内容请参考如下：

ptp_dst_mac     01:80:C2:00:00:0E
network_transport   L2
delay_mechanism     P2P
domainNumber        0
  • 802.1AS Automotive协议

多台激光雷达PTP授时​

在网络中对多台激光雷达同时使用802.1AS协议进行时间同步，需要事先确认所使用的交换机型号是否支持TSN（即时间敏感网络，属于IEEE 802.1的扩展协议栈，旨在以太网协议的数据链路层提供一套通用的时间敏感机制，以确保数据传输的实时性和可靠性）。

  • 对于支持TSN的交换机，则可按照单台激光雷达的PTP授时方法将主机配置为PTP主时钟，并将每台激光雷达的Switch Type参数都设置为'TSN'（如下图所示），即可实现多激光雷达与同一PTP主时钟的时间同步。

  • 对于不支持TSN的普通类型交换机，则无法直接使用标准linuxptp软件程序配置PTP主时钟与多台激光雷达同时进行gPTP时间同步。此时可以参考以下方法先对linuxptp程序中的部分逻辑代码进行调整，使其兼容使用普通交换机的gPTP对时网络：

  1. 在Linuxptp文件夹下运行指令进行主时钟配置并启动授时：

二、硬件设备方式PTP授时​

使用专用的硬件PTP主时钟设备为禾赛激光雷达进行PTP授时，对应的硬件连接方式可参考下图：

该授时方法通常可以获得更高的PTP时间同步精度（< 1μs）。与软件模拟授时方式类似，用户需要根据该硬件设备的操作指导，正确配置相应的PTP协议与主时钟参数并启动授时。（下图展示了Trimble Thunderbolt GM200专用PTP Master设备的参数配置界面）

当PTP主时钟启动授时后，可通过监控从激光雷达中实时接收的点云数据时间戳以及获取到的PTP状态（应为Locked 或 Tracking）信息，判断是否处于正常对时状态；若激光雷达未能与主时钟建立正常授时，则需要检查当前的网络链路硬件连接状态，以及主从时钟双方的PTP参数配置，并重新启动授时进行确认。
